API Login:

GET /api/login - responds with a login page

POST /api/login - responds with json web token if the password is correct
>> expecting { "password": "********" } to be in the body


API Testing:

GET /api - responds with this html page

GET /api/path - responds with the path after /api/

GET /api/time - responds with the server time

GET /api/log - responds with the log.txt file

GET /api/headers - responds with a JSON of the request headers

POST /api/body - responds with posted body (json, urlencoded form, plain text)

GET /api/query?key=value - responds with a JSON of all the key value pairs

GET /api/json - responds with JSON data
